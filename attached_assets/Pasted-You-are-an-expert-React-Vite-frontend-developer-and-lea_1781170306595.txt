Add these lessons with professional content:

Lesson 1:
Structured vs Semi-structured vs Unstructured

Content:
Structured:
- Fixed schema
- Rows and columns
- Tables
- SQL
Keywords:
table, row, column, schema, relational, SQL
Answer:
Structured data / Relational database / Azure SQL Database

Semi-structured:
- Some structure but flexible schema
- JSON, XML
Keywords:
JSON, XML, nested fields, flexible schema, different properties
Answer:
Semi-structured data / Document database / Azure Cosmos DB

Unstructured:
- No fixed schema
- Files, images, videos, audio, PDFs
Keywords:
image, audio, video, binary, file, object
Answer:
Unstructured data / Azure Blob Storage

Rule:
Tables → Structured
JSON/XML → Semi-structured
Images/Videos/Audio → Unstructured

Lesson 2:
OLTP vs OLAP

OLTP:
- Transactional workload
- Frequent small reads/writes
- Normalized data
- Current operational data
Examples:
bank transfer, order, payment, POS system

OLAP:
- Analytical workload
- Read-heavy queries
- Aggregations
- Historical data
- Data warehouse
Examples:
sales dashboard, trends, reports, analytics

Rule:
Order, payment, transfer, POS → OLTP
Reports, trends, warehouse, aggregation → OLAP

Lesson 3:
Batch vs Streaming

Batch:
- Data collected first then processed later
- Schedule or threshold
- Higher latency acceptable
Keywords:
daily, monthly, every 12 hours, after one million records

Streaming:
- Data processed as it arrives
- Low latency
- Real-time events
Keywords:
real-time, sensors, telemetry, rolling average, events as they happen

Rule:
Daily/monthly/scheduled → Batch
Real-time/sensors/rolling average → Streaming

Lesson 4:
ETL vs ELT

ETL:
Extract → Transform → Load
Transform before loading.
Good when cleaning/filtering before destination.

ELT:
Extract → Load → Transform
Transform inside the target system.
Good when the target is powerful like a data warehouse.

Rule:
Transform before load → ETL
Load before transform → ELT

Lesson 5:
Analytics Types

Descriptive:
What happened?

Diagnostic:
Why did it happen?

Predictive:
What will happen?

Prescriptive:
What action should we take?

Cognitive:
AI-based understanding such as speech, text, image recognition.

Rules:
Current or historical chart → Descriptive
Why sales changed → Diagnostic
Forecast future → Predictive
Recommend action → Prescriptive
Recognize speech/image/text entities → Cognitive

Lesson 6:
Data Roles

DBA:
backup, restore, permissions, security, performance, availability

Data Engineer:
pipelines, ingestion, transformation, analytical stores

Data Analyst:
reports, dashboards, charts, business insights

Data Scientist:
machine learning, hidden patterns, prediction models

Rule:
Backup/security/performance → DBA
Pipelines/ingestion/transformation → Data Engineer
Charts/reports/dashboards → Data Analyst
ML/hidden patterns → Data Scientist

Lesson 7:
ACID

Atomicity:
All or nothing.

Consistency:
Valid state before and after transaction.

Isolation:
Transactions do not expose intermediate states.

Durability:
Committed data persists after failure.

Rule:
All-or-nothing → Atomicity
Valid rules → Consistency
Concurrent transaction separation → Isolation
Saved after crash → Durability

Lesson 8:
SQL Language Categories

DDL:
CREATE, ALTER, DROP
Defines structure.

DML:
SELECT, INSERT, UPDATE, DELETE
Works with data.

DCL:
GRANT, REVOKE, DENY
Controls permissions.

Rule:
CREATE table → DDL
INSERT row → DML
GRANT permission → DCL

Lesson 9:
Azure SQL Options

Azure SQL Database:
PaaS
Managed database
Good for cloud apps and SaaS apps.

Azure SQL Managed Instance:
PaaS
Higher SQL Server compatibility
Supports more SQL Server features.

SQL Server on Azure VM:
IaaS
Full control over OS and SQL Server
Best for lift-and-shift with minimal changes.

Rule:
Managed single DB → Azure SQL Database
High compatibility managed → Managed Instance
Full OS control → SQL Server on Azure VM

ACLs
